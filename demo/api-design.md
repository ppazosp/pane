# API Design

## REST Conventions

All endpoints follow `/{resource}/{id?}` pattern. Responses wrapped in:

```json
{
  "data": {},
  "error": null,
  "meta": { "page": 1, "total": 42 }
}
```

### Resourecs

### Users

| Action | Endpoint            | Auth           |
| ------ | ------------------- | -------------- |
| List   | `GET /users`        | Admin          |
| Get    | `GET /users/:id`    | Owner or Admin |
| Update | `PATCH /users/:id`  | Owner          |
| Delete | `DELETE /users/:id` | Admin          |

### Projects

| Action | Endpoint              | Auth          |
| ------ | --------------------- | ------------- |
| List   | `GET /projects`       | Authenticated |
| Create | `POST /projects`      | Authenticated |
| Get    | `GET /projects/:id`   | Member        |
| Update | `PATCH /projects/:id` | Owner         |

## Error Codes

* `AUTH_REQUIRED` — missing or expired token

* `FORBIDDEN` — valid token, insufficient permissions

* `NOT_FOUND` — resource doesn't exist

* `VALIDATION` — request body failed schema check

## Rate Limits

> 100 requests/min per user. 1000/min per IP for unauthenticated routes. Returns `429` with `Retry-After` header.
