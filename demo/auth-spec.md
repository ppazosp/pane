# Auth System Spec

## Overview

Token-based authentication with JWT + refresh tokens. Sessions stored in Supabase, passwords hashed with Argon2.

## Architecture

```
Client -> API Gateway -> Auth Service -> Supabase
                              |
                         Redis (sessions)
```

## Endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/auth/signup` | POST | Create account + send verification email |
| `/auth/login` | POST | Return JWT + refresh token pair |
| `/auth/refresh` | POST | Rotate refresh token |
| `/auth/logout` | DELETE | Revoke session |

## Token Format

```typescript
interface JWTPayload {
  sub: string      // user ID
  email: string
  role: "user" | "admin"
  iat: number
  exp: number      // 15 min
}
```
